SHiNode - Stateless History Node

A lightweight Ethereum history node that syncs directly from P2P. Run your own indexer without paying for RPC services.

GitHub Get Started

Why SHiNode?

Running a full node just to index events is expensive — terabytes of state, constant maintenance through forks, slow sync. Paying for RPC providers costs even more. SHiNode gives you a free, local, unlimited RPC for event indexing without any of that.

  • Fast sync — over 1,000 blocks/sec. Full Mainnet in ~6 hours on SSD, under a day on HDD
  • Runs on anything — low CPU, low memory, works on HDDs. A Mac Mini, a VPS, or that old PC in the closet
  • Small footprint — under 800 GB for full Mainnet, or ~250 GB for events only
  • Partial sync — start from any block, sync only what you need (default: block 10M, Uniswap V2 deployment)
  • Indexer-compatible — plug in rindexer, Ponder (event-only), or any eth_getLogs indexer
  • Zero maintenance — no EVM upgrades, no fork management, no pruning
  • Free local RPC — no rate limits, no API keys, no monthly bills

What It Can Do

  • Sync ETH Mainnet — any block range from P2P, no RPC provider needed
  • Follow the chain head — stay synced with new blocks and handle reorgs
  • Serve event logs — eth_getLogs with filtering by address and topics
  • Run indefinitely — graceful shutdown, restart-safe, resumes where it left off
  • Visualize progress — fullscreen TUI dashboard with real-time stats

Known Limitations

  • Ethereum Mainnet only — L2s don't expose receipts over P2P yet
  • No verification yet — receipt hash verification and CL integration are planned
  • No eth_call — requires EVM state (by design)
  • No traces or state queries — requires execution (by design)

Quick Start

# Build
cargo build --release

# Run (syncs from block 10M to head)
cargo run --release

# Custom range
cargo run --release -- --start-block 18000000 --end-block 18100000
